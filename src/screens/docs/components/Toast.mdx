# Toast

画面の隅に積む通知。 success / error / 進行中などを邪魔せず伝える。 toaster.create() で 1 行追加するだけで、 描画は App root の Toaster が引き受ける imperative API。

## Usage

success / error / warning / info / loading / neutral。 type で icon が切り替わる。

<Example code={ToastTypesSource}>
  <ToastTypesDemo />
</Example>

## Import

```tsx
import { toaster } from "@/components/ui/toast";
```

```tsx
toaster.create({
  title: "Saved",
  description: "Your changes are live.",
  type: "success",
});
```

## Setup

createToaster で singleton を作り、 Toaster component を root に置く。 ui-lab では既にやってある。

```tsx
// components/ui/toast.tsx
export const toaster = createToaster({ placement: "bottom-end", max: 3 });

// App.tsx
import { Toaster } from "@/components/ui/toast";
function App() {
  return (
    <>
      <Routes>...</Routes>
      <Toaster />
    </>
  );
}
```

## With action

Undo パターン。 action.label + action.onClick を渡すと自動で ActionTrigger が並ぶ。

## Promise flow

toaster.promise() で loading → success/error を 1 つの toast で遷移。 fetch のように非同期処理を直接渡せる。

## Anatomy

App root に Toaster を 1 つ mount し、 toaster.create() で imperative に追加する。 個別 toast は内部で Toast.Root + Title + Description + Action + Close で組まれる。
